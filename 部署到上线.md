# 部署到上线 — 按顺序做完即可访问

从本机代码到线上可访问，只需下面几步。**把文中「你的用户名」「你的仓库名」换成你的 GitHub 信息。**

---

## 第一步：在 GitHub 建仓库

1. 打开 **https://github.com/new**
2. **Repository name** 填：`budget-travel`（或你喜欢的名字）
3. **不要**勾选 “Add a README file”
4. 点 **Create repository**

---

## 第二步：在本地推送代码

（可选：想先确认能构建成功，可在项目目录执行 `npm install`，再执行 `npm run build`，无报错即可继续。）

在电脑上打开**终端**（PowerShell 或 CMD），进入项目目录后，**整段复制**下面命令执行。本仓库示例（qysj369 / curqiongyou-shijie-）可直接用：

```bash
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
git init
git add .
git commit -m "feat: 穷游世界上
git branch -M main
git remote add origin https://github.com/qysj369/curqiongyou-shijie-.git
git push -u origin main
```

若用你自己的仓库，把 `qysj369` 和 `curqiongyou-shijie-` 换成你的用户名和仓库名即可。

- 若提示登录：按提示用浏览器登录 GitHub，或使用 [Personal Access Token](https://github.com/settings/tokens) 作为密码。

---

## 第三步：开启 GitHub Pages

1. 打开该仓库页面 → **Settings** → **Pages**
2. **Build and deployment** → **Source** 选择 **GitHub Actions**
3. 保存（无需其他设置）

---

## 第四步：等部署完成

- 推送后会自动触发 **Deploy to GitHub Pages** 工作流
- 到 **Actions** 页签可看进度，约 1～2 分钟变绿即完成
- 在浏览器打开：**https://你的用户名.github.io/你的仓库名/**  
  本仓库示例：**https://qysj369.github.io/curqiongyou-shijie-/**  
  其他示例：`https://zhangsan.github.io/budget-travel/`

**到此网站已部署上线，可正常访问。**

---

## 第五步（可选）：便于网友搜索

上线后建议做两处替换，再提交一次代码：

1. **index.html**  
   把其中的 `YOUR_SITE_URL` 换成你的真实地址（如 `https://你的用户名.github.io/你的仓库名`），保存后执行：
   ```bash
   git add index.html
   git commit -m "chore: 更新站点 URL"
   git push
   ```

2. **public/sitemap.xml**  
   把 `YOUR_USERNAME`、`YOUR_REPO` 全部换成你的用户名和仓库名，保存后同样 `git add`、`commit`、`push`。

---

## 若推送或部署报错

- **推送失败 / 认证失败**：检查用户名、仓库名是否正确；用 Token 时用 Token 当密码。
- **Actions 构建失败**：在仓库 **Actions** 里点失败的那次运行，查看红色报错；常见为 Node 或依赖问题，可把报错贴给开发者排查。
- **打开链接 404**：确认 **Settings → Pages** 里 Source 已选 **GitHub Actions**，且 Actions 里最近一次部署是成功的（绿色勾）。

---

## 上线前：媒体链接与 AI 客服（可选）

### 无缝链接各大媒体 / 视频站

- **分享**：攻略页「分享到」已支持一键打开 **微博、Twitter、QQ 空间** 分享当前页；并提供 **抖音、快手、小红书、B站、YouTube、Instagram** 等入口（复制链接后到对应 App 发布）。
- **关注我们**：在页脚展示「关注我们」及各平台链接。需在 **环境变量** 中配置（构建时生效）：
  - `VITE_LINK_WEIBO`、`VITE_LINK_DOUYIN`、`VITE_LINK_XIAOHONGSHU`、`VITE_LINK_BILIBILI`、`VITE_LINK_YOUTUBE`、`VITE_LINK_TWITTER`、`VITE_LINK_INSTAGRAM`  
  填对应官方主页或账号链接，未配置的不会显示。

### AI 客服 / 智能助手

- **入口**：站点右下角悬浮「AI 小助手」按钮，点击展开对话面板，用户可输入问题。
- **默认**：未接后台时使用**本地模拟回复**（根据关键词回复预算、签证、旅伴等），上线即可用。
- **接入真实后台**：在环境变量中设置 `VITE_AI_API_URL` 为你的 AI 接口地址。接口约定：
  - 请求：`POST`，`Content-Type: application/json`，body `{ "message": "用户输入" }`
  - 响应：JSON，含 `reply` 字段（字符串），即助手回复。
- **关闭 AI 入口**：设置 `VITE_AI_CHAT_ENABLED=false`（或 `0`）后重新构建，右下角入口不再显示。

在 **Vercel / Netlify** 等可在项目设置里添加上述环境变量；**GitHub Pages** 若用 Actions 构建，可在仓库 Settings → Secrets 添加后于 workflow 中通过 `env` 传入（或本地 `BASE=xxx VITE_LINK_WEIBO=xxx npm run build` 再提交 dist）。

## 自定义域名（可选）

适用于：已在域名商（阿里云、腾讯云、Cloudflare 等）购买域名，想让访问用 `www.你的域名.com` 而非 `用户名.github.io/仓库名`。

1. **GitHub 仓库** → `Settings` → `Pages` → **Custom domain** 填你的域名（如 `www.example.com`），保存。按 GitHub 提示在域名商处添加 **CNAME** 或 **A 记录**（GitHub 文档有当前 IP 列表：https://docs.github.com/zh/pages/configuring-a-custom-domain-for-your-github-pages-site ）。
2. **本项目 Vite**：GitHub Pages 根路径若仍是 `/仓库名/`，自定义域名一般指向根 **`https://www.example.com/`**，需在 workflow 的 Build 步骤把 `BASE` 改为 `/`（与仓库 `Settings → Pages` 里自定义域名匹配）。若你固定使用子路径部署，则域名 + 路径要一致，否则会出现资源 404。改完 `.github/workflows/deploy-pages.yml` 后重新推送触发部署。
3. **站内链接**：把 `index.html`、`sitemap.xml` 里的站点 URL 换成你的正式域名后再推送。
