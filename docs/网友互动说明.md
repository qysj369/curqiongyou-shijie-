# 网友互动简评说明

本站支持多种互动方式，无需自建后端。**所有互动均须遵守社区公约：禁止粗俗语言、政治攻击、信仰与宗教歧视等不文明行为。**

---

## 0. 网友留言板（各国）

- 导航栏进入 **网友留言板**，可发帖：**分享旅行美好体验**、**吐槽/避坑**、**其他**。
- 可选填国家/地区，便于按国筛选；每条留言可**点赞**，支持按国家与类型筛选。
- 发帖与点赞数据**仅保存在本机**；如需与全球网友讨论，请使用 GitHub 讨论区（见下文）。
- 发布前会做**不文明用语过滤**，含违规词将提示「请勿使用不文明用语，请修改后重试」。

---

## 1. 我的简评（仅存本机）

每篇攻略底部有「我的简评」输入框，写下的内容**只保存在你当前设备**的浏览器里，不会上传。适合：

- 记录自己的心得、花销、注意事项
- 下次打开同一篇攻略时回顾

数据存在 `localStorage`，清除浏览器数据会丢失。

---

## 2. GitHub 讨论区（可选）

若希望网友在站内看到**所有人**的留言，可接入 **Giscus**（基于 GitHub Discussions）：

1. 在 GitHub 仓库中开启 **Discussions**
2. 安装 [Giscus App](https://github.com/apps/giscus)，并记下 **Repository ID**、**Category**、**Category ID**（在 [giscus 配置页](https://giscus.app/zh-CN) 可查）
3. 在项目根目录新建 `.env`（不要提交到 Git 可加 `.env` 到 `.gitignore`），写入：

```env
VITE_GISCUS_REPO=你的用户名/你的仓库名
VITE_GISCUS_REPO_ID=你的仓库ID
VITE_GISCUS_CATEGORY=Comments
VITE_GISCUS_CATEGORY_ID=你的分类ID
```

4. 重新构建并部署后，攻略详情页会显示 Giscus 评论区，网友用 GitHub 账号即可留言。

若暂不配置，页面会显示温馨提示，并可用「我的简评」做个人备忘。

---

## 3. 讨论区链接（可选）

在 `.env` 中设置：

```env
VITE_GITHUB_DISCUSSIONS_URL=https://github.com/你的用户名/你的仓库名/discussions
```

部署后，攻略页会显示「去 GitHub 讨论区留言」链接，方便网友跳转交流。
