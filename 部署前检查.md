# 部署前检查

上线前按下面做一遍，可减少部署后问题。

---

## 1. 构建是否通过

在项目目录 `budget-travel` 下执行：

```bash
npm install
npm run build
```

无报错、且生成 `dist` 文件夹即表示通过。

---

## 2. 本地预览构建结果

```bash
npm run preview
```

浏览器打开终端里提示的地址（如 http://localhost:4173），逐页点一遍：

- 首页、搜索与筛选
- 目的地列表、排序、点进一个目的地详情
- 攻略列表、排序与筛选、点进一篇攻略、收藏、分享/复制链接/打印/截图
- 我的收藏、面包屑
- 随便打一个错误地址，确认 404 页正常
- 切换中/英文，看文案是否正常

---

## 3. 按《如何部署上线》执行部署

打开 **如何部署上线.md**，按其中 4 步：

1. GitHub 建仓库  
2. 本地 `git init`、`git add`、`git commit`、`git remote`、`git push`  
3. 仓库 Settings → Pages → Source 选 **GitHub Actions**  
4. 等 Actions 跑完后访问：`https://你的用户名.github.io/你的仓库名/`

---

## 4. 部署后自测

- 用手机或无痕窗口打开线上地址，再点一遍主要流程。
- 若使用 GitHub Pages 子路径，确认首页、列表、详情、收藏、分享等链接和资源都正常。

---

**一键检查（仅构建 + 预览）：**

```bash
npm run deploy:check
```

会先执行 `npm run build`，再执行 `npm run preview`，方便你在本机快速验证。
