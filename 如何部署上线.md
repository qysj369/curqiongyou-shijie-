# 如何部署上线

**推荐**：直接按 **部署到上线.md** 从第一步做到第四步，即可完成部署并访问站点。

按下面 4 步做完，你的站点就会上线。

---

## 第 1 步：在 GitHub 创建仓库

1. 打开 **https://github.com/new**
2. **Repository name** 填：`budget-travel`（或你喜欢的名字）
3. **不要**勾选 “Add a README file”
4. 点击 **Create repository**

---

## 第 2 步：把代码推上去

在电脑上打开终端，进入项目目录 `budget-travel`，依次执行（**把 `你的用户名` 和 `你的仓库名` 换成你自己的**）：

```bash
git init
git add .
git commit -m "feat: 穷游世界初版"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

- 若提示登录：按页面提示用浏览器登录 GitHub，或使用 [Personal Access Token](https://github.com/settings/tokens) 作为密码。

---

## 第 3 步：开启 GitHub Pages

1. 打开你的仓库页面，点 **Settings**
2. 左侧点 **Pages**
3. 在 **Build and deployment** 里，**Source** 选择 **GitHub Actions**
4. 无需其他设置，保存即可

---

## 第 4 步：等部署完成并访问

- 推送代码后会自动触发部署；或到仓库 **Actions** 页签，点 **Run workflow** 手动跑一次
- 约 1～2 分钟后，在浏览器打开：

  **https://你的用户名.github.io/你的仓库名/**

  例如仓库名是 `budget-travel`：**https://你的用户名.github.io/budget-travel/**

---

## 部署后可选（便于网友搜索）

- **站点的真实地址**：打开 `index.html`，把其中的 `YOUR_SITE_URL` 换成你的完整站点地址（如 `https://你的用户名.github.io/你的仓库名`），再提交推送，有利于搜索引擎收录。
- **Sitemap**：把 `public/sitemap.xml` 里的 `YOUR_USERNAME`、`YOUR_REPO` 改成你的，保存后再次提交推送。
- **其他平台**：想用 Vercel / Netlify 时，可看 **README.md** 的「部署」章节。

更多细节见 **DEPLOY-STEP2.md**。
