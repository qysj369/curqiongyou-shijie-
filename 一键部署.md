# 一键部署上线（零经验版）

按下面顺序做，**每一步做完再做下一步**，就能顺利上线。

---

## 第一步：确认本机已安装

1. **Node.js**  
   - 打开 PowerShell 或 CMD，输入：`node -v`  
   - 若显示版本号（如 `v20.x.x`）说明已安装；若提示“不是内部或外部命令”，请先安装：https://nodejs.org/ 选 LTS 版。

2. **Git**  
   - 输入：`git --version`  
   - 若显示版本号说明已安装；若没有，请安装：https://git-scm.com/download/win  

3. **GitHub 账号**  
   - 已注册账号 **qysj369**，并已创建仓库 **curqiongyou-shijie-**（若还没创建，打开 **https://github.com/new** 创建，仓库名填 `curqiongyou-shijie-`，不要勾选 Add README）。

---

## （重要）首次在本机用 Git：先配置姓名和邮箱

若执行 `git commit` 时出现 **Please tell me who you are**，必须先做一次（邮箱可改成你的；也可用 GitHub 的 noreply 邮箱）：

```powershell
git config --global user.name "你的名字或昵称"
git config --global user.email "你的邮箱@example.com"
```

**常见错法：** 在提示符后面**只输入**姓名或 GitHub 用户名（例如单独一行 `魏东` 或 `qysj369`），PowerShell 会报「无法识别」。正确做法是**整行**使用上面的 `git config`，姓名写在**引号里**。

然后再 `git add` / `git commit` / `git push`。

**注意：** `git push -u origin main` 必须**单独一行**，后面**不要**再跟路径或别的字；若出现 **invalid refspec 'c:\Users\...'**，说明把文件夹路径和 `git push` 粘在同一行了，请分开执行。

---

## 第二步：本地先试构建（避免推送后报错）

在 PowerShell 或 CMD 里执行：

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
npm install
npm run build
```

- 若最后出现 `built in xxx ms` 且没有红色报错，说明构建成功，可以继续。
- 若有报错，先不要推送，把完整报错复制下来找人排查。

---

## 第三步：一键推送到 GitHub

**方式 A：复制整段命令执行（推荐）**

1. 复制下面**整段**（从 `cd` 到 `git push`）。
2. 在 PowerShell 或 CMD 里**右键粘贴**，回车。
3. 若提示登录 GitHub：按提示在浏览器里登录或使用 Personal Access Token 当密码。

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
git init
git add .
git commit -m "feat: 穷游世界上线"
git branch -M main
git remote add origin https://github.com/qysj369/curqiongyou-shijie-.git
git push -u origin main
```

**注意（PowerShell）：** 不要只输入 **`git commit`**（会打开 Vim，容易提示 *Output is not to a terminal* 并满屏乱码）。请始终使用 **`git commit -m "说明"`**。若已乱屏，关掉这个终端窗口，新开一个再操作。可选：把默认编辑器改成记事本，以后即使误开编辑器也好关：

```powershell
git config --global core.editor notepad
```

**方式 B：双击脚本**

- 直接双击项目里的 **`一键部署.bat`**，按提示操作（若已执行过 `git init` 且已有 remote，脚本会只做 add、commit、push）。

---

## 第四步：在 GitHub 上开启 Pages

1. 打开浏览器，进入：**https://github.com/qysj369/curqiongyou-shijie-**
2. 点顶部 **Settings**（仓库设置）。
3. 左侧找到 **Pages**。
4. 在 **Build and deployment** 里：
   - **Source** 选 **GitHub Actions**（不是 “Deploy from a branch”）。
5. 不用点 Save，选完就会生效。

---

## 第五步：等部署完成并访问

1. 点仓库顶部的 **Actions**。
2. 会看到一条 **Deploy to GitHub Pages** 在工作，等约 1～2 分钟，前面出现绿色勾 ✅ 即表示部署成功。
3. 在浏览器打开：  
   **https://qysj369.github.io/curqiongyou-shijie-/**  

若打不开，等 1 分钟再试；若仍不行，确认 Actions 里是绿色勾且 Settings → Pages 里 Source 选的是 **GitHub Actions**。

---

## 以后更新网站怎么操作

改完代码后，在项目目录执行：

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
git add .
git commit -m "更新内容"
git push
```

推送后 Actions 会自动重新构建并更新网站，几分钟后刷新 **https://qysj369.github.io/curqiongyou-shijie-** 即可看到最新版。

## 已绑定自定义域名后（重要）

若你已经绑定了自定义域名（如 **www.穷游世界.com**），请注意：

1. 正式访问地址优先用：**https://www.穷游世界.com**（`github.io` 链接可作为备用）。
2. GitHub → Settings → Pages 中，`Enforce HTTPS`（强制执行 HTTPS）可勾选时请勾上。
3. 若域名打开白屏，通常是构建基路径不对。确保工作流里构建环境变量是：

```yml
BASE: /
```

（不要使用 `/${{ github.event.repository.name }}/`，那是给 `github.io/仓库名/` 子路径站点用的。）

## 常见问题

| 情况 | 处理 |
|------|------|
| `Please tell me who you are` / commit 失败 | 先执行上面的 **git config --global user.name / user.email**（整行复制，不要只打姓名），再重新 `git commit`。 |
| `Vim: Output is not to a terminal` / 满屏乱码 | 说明执行了**不带** `-m` 的 `git commit`，Git 调起了 Vim。关闭该终端，新开 PowerShell，改用 **`git commit -m "feat: 穷游世界上线"`**；可选执行 **`git config --global core.editor notepad`**。 |
| 输入姓名后报「无法识别」| 说明只打了名字，没有打 `git config --global user.name "..."` 这一整行。 |
| `src refspec main does not match any` / push 失败 | 常见原因：① 本地还没有成功 `commit`，或分支不叫 `main` —— 执行 `git status`、`git branch`；需要时 `git branch -M main` 再推。② **行尾多打了 `;` 或中文分号 `；`**（或复制进了全角字符），Git 会把分支名弄错 —— 请只输入 **`git push -u origin main`** 然后回车，**main 后面不要任何符号**。 |
| `invalid refspec 'c:\Users\...'` | `git push -u origin main` 必须单独一行，勿与 `cd` 或路径写在同一行；分开逐行执行。 |
| `Recv failure` / `Connection was reset` 推送失败 | 访问 GitHub 的网络被中断。可多试几次 `git push`；换手机热点；有代理时给 Git 配置 `http.proxy` / `https.proxy`（见下）。 |
| `Failed to connect to github.com port 443` / 超时 | **完全连不上** GitHub。先在浏览器打开 **https://github.com** — 若也打不开，说明是当前网络限制，需换 **手机热点**、或开启 **VPN/代理** 后再 `git push`；若浏览器能开但 Git 不行，同样要给 Git 配代理（见下）。 |
| `git push` 提示 403 / 认证失败 | 检查用户名、仓库名是否写对；若用 HTTPS，在浏览器按提示登录，或到 GitHub → Settings → Developer settings → Personal access tokens 新建 Token，用 Token 当密码。 |
| `git remote add origin` 报错 “already exists” | 说明之前加过了，可跳过这行，直接执行 `git push -u origin main`。 |
| Actions 里构建失败（红色叉） | 点进该次运行，看是哪一步报错；常见是 Node 版本或依赖问题，把红色报错复制下来排查。 |
| 打开链接是 404 | 确认 Settings → Pages 里 Source 选的是 **GitHub Actions**，且 Actions 里最近一次是绿色勾。 |

### Git 走代理（浏览器能上 GitHub、但 `git push` 仍失败时）

代理软件（如 Clash）里记下 **HTTP 端口**（常见 `7890`），在 PowerShell 执行（端口按你的软件改）：

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

再执行 `git push -u origin main`（**末尾不要加分号**）。不用代理后取消：

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```
