# 一键部署上线（零经验版）

按下面顺序做，**每一步做完再做下一步**，就能顺利上线。

---

## 第一步：确认本机已安装

1. **Node.js**  
   - 打开 PowerShell 或 CMD，输入：`node -v`  
   - 若显示版本号（如 `v20.x.x`）说明已安装；若提示“不是内部或外部命令”，请先安装：https://nodejs.org/ 选 LTS 版。

2. **Git**  
   - 输入：`git --version`  
   - 若显示版本号说明已安装；若没有，请安装：https://git-scm.com/download/win  

3. **GitHub 账号**  
   - 已注册账号 **qysj369**，并已创建仓库 **curqiongyou-shijie-**（若还没创建，到 himage.png 创建，仓库名填 `curqiongyou-shijie-`，不要勾选 Add README）。

---

## 第二步：本地先试构建（避免推送后报错）

在 PowerShell 或 CMD 里执行：

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
npm install
npm run build
```

- 若最后出现 `built in xxx ms` 且没有红色报错，说明构建成功，可以继续。
- 若有报错，先不要推送，把完整报错复制下来找人排查。

---

## 第三步：一键推送到 GitHub

**方式 A：复制整段命令执行（推荐）**

1. 复制下面**整段**（从 `cd` 到 `git push`）。
2. 在 PowerShell 或 CMD 里**右键粘贴**，回车。
3. 若提示登录 GitHub：按提示在浏览器里登录或使用 Personal Access Token 当密码。

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
git init
git add .
git commit -m "feat: 穷游世界上线"
git branch -M main
git remote add origin https://github.com/qysj369/curqiongyou-shijie-.git
git push -u origin main
```

**方式 B：双击脚本**

- 直接双击项目里的 **`一键部署.bat`**，按提示操作（若已执行过 `git init` 且已有 remote，脚本会只做 add、commit、push）。

---

## 第四步：在 GitHub 上开启 Pages

1. 打开浏览器，进入：*image.pngimage.pngimage.pngimage.pngimage.pngimage.pngimage.png**
2. 点顶部 **Settings**（仓库设置）。
3. 左侧找到 **Pages**。
4. 在 **Build and deployment** 里：
   - **Source** 选 **GitHub Actions**（不是 “Deploy from a branch”）。
5. 不用点 Save，选完就会生效。

---

## 第五步：等部署完成并访问

1. 点仓库顶部的 **Actions**。
2. 会看到一条 **Deploy to GitHub Pages** 在工作，等约 1～2 分钟，前面出现绿色勾 ✅ 即表示部署成功。
3. 在浏览器打开：  
   **https://qysj369.github.io/curqiongyouimage.png-shijie-/**  

若打不开，等 1 分钟再试；若仍不行，确认 Actions 里是绿色勾且 Settings → Pages 里 Source 选的是 **GitHub Actions**。

---

## 以后更新网站怎么操作

改完代码后，在项目目录执行：

```powershell
cd "c:\Users\Administrator\Desktop\新建文件夹 (5)\budget-travel"
git add .
git commit -m "更新内容"
git push
```

推送后 Actions 会自动重新构建并更新网站，几分钟后刷新 **https://qysj369.github.io/curqiongyou-shijie-** 即可看到最新版。

## 常见问题

| 情况 | 处理 |
|------|------|
| `git push` 提示 403 / 认证失败 | 检查用户名、仓库名是否写对；若用 HTTPS，在浏览器按提示登录，或到 GitHub → Settings → Developer settings → Personal access tokens 新建 Token，用 Token 当密码。 |
| `git remote add origin` 报错 “already exists” | 说明之前加过了，可跳过这行，直接执行 `git push -u origin main`。 |
| Actions 里构建失败（红色叉） | 点进该次运行，看是哪一步报错；常见是 Node 版本或依赖问题，把红色报错复制下来排查。 |
| 打开链接是 404 | 确认 Settings → Pages 里 Source 选的是 **GitHub Actions**，且 Actions 里最近一次是绿色勾。 |
